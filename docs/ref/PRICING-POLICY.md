## 5. 제한 정책

### 하드 리밋 (크레딧 외 추가 제한)
- **동시 generation_jobs**: Free 1건 / Starter 3건 / Pro 5건 / Business 10건
- **단일 프로젝트 최대 블록 수**: 50
- **업로드 파일 크기**: Free 5MB / 유료 20MB
- **API Rate Limit**: Free 10req/min / Starter 30req/min / Pro 60req/min / Business 120req/min

### 크레딧 소진 시 동작
1. 크레딧 잔량 부족 → API 응답 `402 Payment Required` + 잔여 크레딧 정보
2. UI에서 크레딧 부족 모달 표시 → 추가 구매 또는 업그레이드 유도
3. 이미 진행 중인 job은 완료까지 허용 (중간 중단 방지)

### 월 초기화
- 매월 구독 결제일 기준 크레딧 리셋
- 미사용 월 크레딧은 이월 불가
- 추가 구매 크레딧은 이월됨

## 6. 스키마 호환성 분석

### 기존 스키마와의 관계

| 기존 테이블 | 역할 | 가격 정책 연동 |
|------------|------|--------------|
| `workspaces` | 과금 단위 | 구독은 workspace 단위로 연결 |
| `usage_ledger` | 사용 기록 | 크레딧 차감 기록에 재활용 |
| `generation_jobs` | 작업 추적 | 동시 job 수 제한 체크에 활용 |
| `projects` | 프로젝트 | 프로젝트 수 제한 체크에 활용 |
| `compose_templates` | 템플릿 | 템플릿 수 제한 체크에 활용 |

### 추가 필요 테이블

```
profiles              — Supabase Auth 연동 (auth.users.id 참조)
subscriptions         — workspace별 구독 상태
credit_balances       — workspace별 크레딧 잔고
credit_transactions   — 크레딧 차감/충전 이력
pricing_tiers         — 티어 정의 (seed data)
```

### usage_ledger 확장
- 기존 `usage_ledger`에 `credits_consumed` 컬럼 추가
- 기존 `cost_estimate`는 내부 원가 추적용으로 유지
- `credits_consumed`는 사용자 과금용 크레딧 차감량
